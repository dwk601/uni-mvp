## Key Design Principles

### 1. Normalization
- **3NF (Third Normal Form)** compliance
- Minimal data redundancy
- Clear entity separation

### 2. Performance
- Strategic indexing on frequently queried columns
- Partial indexes with WHERE clauses for efficiency
- GIN index for JSONB data

### 3. Data Integrity
- Foreign key constraints
- UNIQUE constraints on natural keys
- NOT NULL constraints on critical fields

### 4. Temporal Design
- Year-based time series data
- Allows tracking changes over time
- UNIQUE constraints on (unitid, year)

### 5. Flexibility
- JSONB for semi-structured data (admission documents)
- Text fields for varying-length content
- Nullable fields for optional data

### 6. Security
- Row Level Security (RLS) enabled
- Role-based access control
- Read-only by default for public API

### 7. API-First Design
- RESTful endpoints for all tables
- Custom functions for complex queries
- Views for common data combinations
- PostgREST auto-generates OpenAPI docs
